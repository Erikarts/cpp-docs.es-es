# <a name="nolock-functions"></a>_nolock (Funciones)

Se trata de funciones que no tienen que realizar ningún bloqueo. Se proporcionan para los usuarios que necesitan obtener el máximo rendimiento. Para más información, vea [Rendimiento de bibliotecas multiproceso](../c-runtime-library/multithreaded-libraries-performance.md).

Utilice las funciones _nolock solo si el programa es realmente de un único subproceso o si realiza su propio bloqueo.
